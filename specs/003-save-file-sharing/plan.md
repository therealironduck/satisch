# Implementation Plan: Save File Sharing

**Branch**: `003-save-file-sharing` | **Date**: 2026-05-16 | **Spec**: [spec.md](spec.md)  
**Input**: Feature specification from `specs/003-save-file-sharing/spec.md`

## Summary

Enable save game owners to share their saves with other registered users via email invite. Invited users gain full edit access (once factories are built) and can see shared saves in a "Shared with me" dashboard filter. Owners can revoke access; collaborators can leave. All enforced via Supabase RLS and a SECURITY DEFINER RPC for the invite flow.

## Technical Context

**Language/Version**: TypeScript / Vue 3 (Nuxt 4)  
**Primary Dependencies**: `@nuxt/ui` v4, `@supabase/supabase-js`, Tailwind CSS v4

**Testing**: Manual — `bun run dev` golden-path validation; `bun run typecheck` + `bun run lint` must pass  
**Target Platform**: Browser SPA (SSR disabled)  
**Project Type**: Web application  
**Performance Goals**: Invite action completes in under 30 seconds; collaborator list is reactive via Supabase Realtime  
**Constraints**: SPA-only; no server routes; all backend logic via Supabase RLS + RPC; MDI icons only  
**Scale/Scope**: No collaborator cap; dashboard filter is client-computed from the already-fetched saves list

## Implementation Phases

### Phase A: Database (Migration)

Single migration file: `supabase/migrations/20260516000001_add_save_game_collaborators.sql`

Contents:

1. Create `save_game_collaborators` table with cascade deletes and UNIQUE constraint
2. Index on `(save_game_id)` and `(user_id)`
3. Drop existing `save_games_select_own` policy; create new `save_games_select` allowing owner OR collaborator
4. Enable RLS on `save_game_collaborators`; create all four policies (select_owner, select_self, insert_deny, delete_owner, delete_self)
5. Create `invite_save_game_collaborator` RPC (SECURITY DEFINER)
6. Grant table and function access to `authenticated` role

### Phase B: Types & Composables

**`app/types/saveGame.ts`** — add `SaveGameCollaborator` interface.

**`app/composables/useSaveGames.ts`** — two changes:

1. `useSaveGames` already fetches all saves visible to the user (RLS now includes shared saves automatically after the migration). No query change needed. Add a realtime channel subscribed to `save_game_collaborators` filtered by `user_id = currentUserId` that triggers `refetch()` on INSERT or DELETE.
2. Reset `fetched = false` on the realtime event so a full re-fetch occurs.

**`app/composables/useSaveGameCollaborators.ts`** — new composable:

- Accepts `saveGameId` as a parameter (or ref)
- Fetches collaborators from `save_game_collaborators` ordered by `added_at`
- Exposes: `collaborators`, `loading`, `error`, `invite(email)`, `remove(collaboratorId)`, `leave(collaboratorId)`
- Subscribes to realtime changes on `save_game_collaborators` filtered by `save_game_id = <id>` for live list updates in the share dialog
- `invite(email)`: calls RPC `invite_save_game_collaborator`, maps error strings to display messages
- `remove(id)`: deletes the collaborator row (owner flow); optimistic removal with rollback
- `leave(id)`: same delete operation, different label (collaborator self-removal)

### Phase C: Dashboard Filter

**`app/pages/dashboard.vue`** — add `activeFilter` ref (`'all' | 'owned' | 'shared'`). Compute `filteredSaves`:

- `all`: all saves
- `owned`: `saves.filter(s => s.owner_id === user.value?.id)`
- `shared`: `saves.filter(s => s.owner_id !== user.value?.id)`

Pass `activeFilter` and `@filter-change` to `DashboardSidebar`. Pass `filteredSaves` (instead of `saves`) to `DashboardSaveList`.

**`app/components/Dashboard/DashboardSidebar.vue`** — rework:

- Accept `saves` and `activeFilter` props; emit `filter-change`
- Add active styling to whichever filter button matches `activeFilter`
- "Shared with me" button: count = `saves.filter(s => s.owner_id !== user.value?.id).length`
- "Owned by me" button: count = `saves.filter(s => s.owner_id === user.value?.id).length`
- "All saves" button remains, count = `saves.length`

**`app/components/Dashboard/DashboardSaveList.vue`** — no functional change needed (already renders `saves` prop). Empty-state message should account for active filter (e.g., "No shared saves yet" vs "No saves yet").

### Phase D: Save Card — Role Badge & Permission Gates

**`app/components/Dashboard/DashboardSaveCard.vue`**:

- Inject `useSupabaseUser()` to get current user id
- Compute `isOwner = save.owner_id === user.value?.id`
- Replace hardcoded `OWNER` badge with conditional: `isOwner ? 'OWNER' : 'SHARED'` (different colors — e.g., `amber` for owner, `neutral` for shared)
- Gate the dropdown menu items: Rename and Delete are only shown when `isOwner`
- Add a "Share" button (visible to all who have access) that opens `SaveGameShareModal`
- Wire `SaveGameShareModal` with `v-if="showShare"` + `v-model:open`

### Phase E: Share Modal

**`app/components/SaveGame/SaveGameShareModal.vue`** — new component:

Structure (using `UModal`):

- **Header**: Save game name + "Share" title
- **Body**:
  - If `isOwner`: Email input (`UInput`) + "Add" button (`UButton`); error alert (`UAlert`) below if invite failed
  - Collaborator list: owner row at top (labeled "OWNER", no remove button), then each collaborator row showing `email` + `added_at` date + remove button (if `isOwner`) or "Leave" button (if current user's own row and not owner)
  - Empty state: "No collaborators yet" when only the owner row is shown
- **Footer**: Close button

Behavior:

- On open: fetch collaborators via `useSaveGameCollaborators(saveGameId)`; realtime channel keeps list live
- Invite submit: call `invite(email)`, show error from RPC response if `success: false`, clear input on success
- Remove/Leave: call `remove(id)` / `leave(id)` with optimistic update
